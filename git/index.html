No time to waste

We will:

  1. Create a repository
    git init
    git clone <remote> <local path>
  2. Add a remote repository
    git remote add <remote_name> <server>
  3. Create a local branch
    git branch <branch_name>
  4. Create a remote branch
    git push <remote_name> <branch_name>
  5. Deleting branch
    git push <remote_name> :<branch_name>
    git branch -d <branch_name>
  6. learn how to avoid conflicts... and how to solve them!
    git checkout <branch_name>
    git branch <merge_savepoint>
    git merge <branch_name>
    Things go wrong?
    git reset --hard <merge_savepoint>
    Things go well?
    git branch -d <merge_savepoint>

Cool! We made it and have some more time...

  1. More conflict solving strategies
    git checkout --theirs -- <file>
    git checkout --ours -- <file>
    https://github.com/git/git/blob/master/Documentation/merge-strategies.txtfile:///home/handrus/Documents/projects/personal/talks/git/git/index.html
  2. More ways to merge
    git cherry-pick <commit>
    git rebase <branch_name>
    git reset --hard <remote_name>/<branch_name>
  3. Using stashes (git stash)
    git stash save <stash_description>
    git stash list
    git stash apply <stash_NUMBER>
    git stash pop
    git stash drop <stash_NUMBER>
    git stash clear
  4. Improving your command line
    git config --global alias.<alias_name> "<comand>"
    git config --global alias.st "status -s"
    git config --global alias.undo "reset --hard HEAD"
    git diff --word-diff
    git checkout -t origin/feature
    git show :/^Merge
  5. git tag
    git tag -a v1.0.0 -m "Creating the first official version."
    git describe --tags
    touch test; git add test; git commit -am "Adding test files"
    git describe --tags
    git push --tags
  6. git clean
    git clean -n -d -x <folder>
    git clean -f -d -x <folder>
